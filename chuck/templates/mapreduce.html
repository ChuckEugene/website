MapReduce Server

Overview:

  • The MapReduce model is a method of processing large amounts of data in parallel on multiple machines

  • Data is first distributed to different systems where it is "mapped", which could be a sorting or filtering function

  • The resultant data files are then "reduced" or summarized in some way

Methodology:

  • Two modules were made: master and worker. The master is a process that spawns threads which are the workers

  • The masters' job is to partition data for the workers, specify which phase of the MapReduce method is being performed, and return a single output

  • The workers perform the functions in a map executable, and subsequently a reduce executable

Example:

  • The server is started up


  • The input parameters of the MapReduce are passed


  • Mapping phase: the workers recieve their instructions


  • Reduce phase: the workers start summarizing the data


  • The output is complete and the master and workers can be killed
